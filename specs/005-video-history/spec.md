# Feature Specification: 動画視聴履歴

**Feature Branch**: `005-video-history`

**Created**: 2026-06-24

---

## Overview

ユーザーが過去に読み込んだ YouTube 動画の履歴を自動的に保存し、履歴一覧から動画を再選択できるようにする。これにより、同じ動画を繰り返し練習・視聴するユーザーが URL を再入力する手間をなくし、作業効率を高める。

---

## Clarifications

### Session 2026-06-24

- Q: 履歴一覧の UI エントリポイントはどこに配置するか → A: ヘッダーに履歴アイコンを追加し、タップでドロワーまたはモーダルを展開
- Q: タイトル・サムネイルの取得タイミングはいつか → A: 動画読み込み完了時点の情報をそのまま保存し、後からの更新はしない。取得できなければ URL を代替表示
- Q: 異なる URL フォーマットの同一動画を重複と判定するか → A: YouTube 動画 ID を抽出して deduplication キーとする。`url` フィールドにはユーザーが入力した URL をそのまま保存し正規化しない
- Q: ストレージへの書き込み失敗時の挙動はどうするか → A: サイレントに継続する（履歴保存失敗時もプレイヤーは通常通り動作し、エラーを表示しない）
- Q: 履歴アイテムの削除操作 UI はどのパターンにするか → A: 各アイテムにゴミ箱アイコンボタンを常時表示する

---

## Problem Statement

現状、LoopTube では毎回 YouTube の URL を手動入力しなければならない。ユーザーが複数の動画を定期的に練習する場合、URL を記憶・コピー・再入力するコストが積み重なり、使い勝手を損なっている。

---

## Goals

- 動画を読み込むたびに自動で履歴へ追加する
- 履歴一覧から 1 タップで動画を再読み込みできるようにする
- 履歴は永続的に保持し、アプリを閉じても消えない
- モバイルでも快適に操作できる UI を提供する

## Non-Goals

- 複数デバイス間での履歴同期（ログイン機能実装後の将来課題）
- 履歴のフォルダー分類・タグ付け
- 再生回数などの統計情報の記録

---

## User Scenarios & Testing

### シナリオ 1: 動画を読み込んだとき自動的に履歴に追加される

**Given** ユーザーが YouTube URL を入力して動画を読み込む  
**When** 動画の読み込みが完了する  
**Then** その動画が履歴の先頭に追加される  
**And** 既に同じ動画が履歴に存在する場合は重複して追加されず、先頭へ移動する

### シナリオ 2: 履歴一覧から動画を選択する

**Given** ユーザーが履歴一覧を開く  
**When** 一覧から任意の動画をタップする  
**Then** その動画が読み込まれ、再生準備が整った状態になる

### シナリオ 3: 履歴からアイテムを削除する

**Given** ユーザーが履歴一覧を開く  
**When** 特定のアイテムを削除する操作を行う  
**Then** そのアイテムが一覧から消える

### シナリオ 4: アプリ再起動後も履歴が残る

**Given** ユーザーが複数の動画を読み込んで履歴に追加した後、ブラウザを閉じる  
**When** 再度アプリを開く  
**Then** 閉じる前の履歴がそのまま表示される

### シナリオ 5: 履歴が上限件数を超えた場合

**Given** 履歴が最大保持件数（50 件）に達している  
**When** 新しい動画を読み込む  
**Then** 最も古い履歴アイテムが自動的に削除され、新しいアイテムが先頭に追加される

### シナリオ 6: 履歴が空のとき

**Given** 履歴にアイテムが 1 件もない  
**When** ユーザーが履歴一覧を開く  
**Then** 「履歴はまだありません」といった空状態メッセージが表示される

## Functional Requirements

### FR-1: 自動履歴追加

- 動画の読み込みが成功するたびに、その動画情報（タイトル・サムネイル・URL）が履歴に自動追加される
- タイトル・サムネイルは動画読み込み完了時点で取得できた値をそのまま保存し、後からの非同期更新は行わない
- タイトルが取得できない場合は空文字を保存する。サムネイルは動画 ID から常に確定的に生成する
- 同一動画の判定には YouTube 動画 ID を使用する。入力 URL の形式（短縮 URL、通常 URL など）に関わらず、同じ動画 ID を持つ場合は重複と見なして先頭へ移動する

### FR-2: 履歴の永続保存

- 履歴はアプリ終了後も保持される
- 未認証ユーザーの場合はデバイスのローカルストレージに保存する
- 最大 50 件を保持し、超過分は最古のアイテムを自動削除する

### FR-3: 履歴一覧の表示

- 直近の視聴順（新しい順）で一覧表示する
- 各アイテムにはサムネイル・タイトルを表示する
- タイトルが取得できない場合は URL を代替表示する
- 履歴が空の場合は空状態メッセージを表示する

### FR-4: 履歴からの動画読み込み

- 一覧のアイテムをタップすると、その動画を現在のプレイヤーに読み込む
- 読み込み後、そのアイテムは履歴の先頭に移動する

### FR-5: 履歴アイテムの削除

- 各アイテムにゴミ箱アイコンボタンを常時表示し、タップで即座に削除できる
- 削除は即座に反映され、永続ストレージにも同期される
- 削除の確認ダイアログは表示しない（誤削除のリスクより操作のシンプルさを優先）

### FR-6: i18n 対応

- 履歴 UI に表示されるすべてのラベル・メッセージは日本語（ja）と英語（en）に対応する

---

## Success Criteria

- 動画読み込み後、1 秒以内に履歴一覧へのアイテム追加が反映される
- モバイル画面（375px 幅）で履歴一覧の全操作（選択・削除）が快適に行える
- ブラウザを閉じて再度開いても 100% の確率で履歴が復元される
- 履歴一覧を開くまでの操作が 2 タップ以内で完了する
- 空状態・上限超過・タイトル未取得のすべてのエッジケースでエラーが発生しない
- ストレージへの書き込み・読み込みが失敗した場合、プレイヤーの動作には影響を与えずサイレントに継続する

---

## Key Entities

### HistoryItem

| フィールド   | 説明                                        |
|--------------|---------------------------------------------|
| id           | YouTube 動画 ID（deduplication キー）        |
| url          | 動画の URL（最後に使用された入力 URL をそのまま保存） |
| title        | 動画タイトル（取得できない場合は空文字）    |
| thumbnailUrl | サムネイル URL（動画 ID から常に生成）      |
| addedAt      | 履歴に追加された日時                        |

---

## Dependencies

- 動画タイトル・サムネイルを取得するための外部情報取得手段（既存の動画読み込み処理に依存）
- 永続ストレージへのアクセス（専用の `HistoryPort` 抽象を新規作成し、既存の Port/Adapter パターンを踏襲する）

---

## Assumptions

- 初期実装は未認証ユーザーのみを対象とし、ローカルストレージに保存する
- デバイス間同期は今回のスコープ外とする
- 最大保持件数は 50 件とする（変更可能な設定項目にはしない）
- 履歴一覧の UI エントリポイントは、ヘッダーに専用の履歴アイコンを追加し、タップでドロワーまたはモーダルを展開する形式とする
- サムネイルはデータモデル上常に存在するが、ブラウザの画像ロードが失敗した場合（ネットワークエラー等）はプレースホルダー SVG を UI フォールバックとして表示する

---

## Out of Scope

- ログイン認証・デバイス間同期
- 履歴のエクスポート・インポート
- フォルダー分類・タグ付け・検索
- 再生履歴の統計（視聴回数、最終視聴時刻など）
